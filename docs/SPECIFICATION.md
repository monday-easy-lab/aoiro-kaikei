# 青色申告かんたん会計 — 仕様書

**バージョン:** 1.1.0  
**最終更新:** 2026年6月  
**対象ユーザー:** 個人事業主（事業所得のみ）  
**対象制度:** 青色申告（65万円 / 55万円 / 10万円控除）

---

## 1. コンセプト

> 会計ソフトを開いた瞬間、「複式簿記」「借方・貸方」という文字を見て、そっと閉じたことがある人へ。

### 設計原則

- **借方・貸方を見せない** — ユーザーは「何をしたか」「どう払ったか」だけ選ぶ
- **完全オフライン** — データはブラウザの localStorage のみ。サーバー送信ゼロ
- **裏側は正しい複式簿記** — 出力される仕訳帳・PL・BSは税理士が確認可能な水準
- **0円で始められる** — 元入金が不明でも開始可能

## 4. かんたん入力の仕様

### 4.1 入力フロー

```
ステップ1: カテゴリ選択（6カテゴリ）
  💰 売上  /  🛒 経費  /  🏦 資金移動
  🏧 生活費  /  🖥️ 固定資産  /  📊 決算

ステップ2: テンプレート選択
  カテゴリ内のテンプレートカードを選択

ステップ3: 金額入力
  日付 / 金額 / 支払方法（該当時）/ メモ
```

### 4.2 支払方法と生成される仕訳

経費・固定資産購入時に表示される。5択。

| 支払方法 | 貸方科目 | 仕訳例（消耗品費の場合） |
|---|---|---|
| 🏦 銀行振込 | 普通預金 (102) | 消耗品費 / 普通預金 |
| 💵 現金 | 現金 (101) | 消耗品費 / 現金 |
| 💳 クレカ | クレジットカード (205) | 消耗品費 / クレジットカード |
| 📱 電子マネー | 電子マネー (107) | 消耗品費 / 電子マネー |
| 🙋 個人立替 | 事業主借 (203) | 消耗品費 / 事業主借 |

### 4.3 支払方法が表示されないケース

- 売上テンプレート（入金先が固定）
- 資金移動テンプレート（科目が固定）
- 生活費テンプレート（科目が固定）
- `cashless: true` のテンプレート（減価償却等、お金の移動がない処理）

### 4.4 注意書きの自動表示

| 条件 | 表示内容 |
|---|---|
| 💳 クレカ選択時 | 「引落日に再入力する必要はありません。引落時は『資金移動→クレカの引き落とし』で処理します。」 |
| 🙋 個人立替選択時 | 「個人のお金で事業の支払いをした場合に選択します。個人口座と事業口座が同じ場合は『銀行振込』を選んでください。」 |
| 減価償却テンプレート選択時 | 「減価償却はお金の移動がない帳簿上の処理です」 |

## 5. 支払方法とクレジットカードの仕訳フロー

### 5.1 クレジットカードの仕訳サイクル

```
① 経費発生時（かんたん入力で「💳 クレカ」を選択）
   ソフトウェア利用料 1,000 / クレジットカード 1,000

② 引落日（かんたん入力「資金移動→クレカの引き落とし」）
   クレジットカード 1,000 / 普通預金 1,000

→ クレカ残高が0に戻る
```

### 5.2 個人立替（事業主借）のフロー

```
① 個人カードで事業経費を支払った時
   ソフトウェア利用料 1,000 / 事業主借 1,000

→ 年度末に事業主借は元入金に自動振替される（年度締め機能）
→ 入力はこの1回のみ。追加の決済処理は不要
```

### 5.3 電子マネーの仕訳サイクル

```
① チャージ時（かんたん入力「資金移動→電子マネーにチャージ」）
   電子マネー 10,000 / 普通預金 10,000

② 経費支払時（かんたん入力で「📱 電子マネー」を選択）
   旅費交通費 500 / 電子マネー 500
```

## 6. 家事按分の仕様

### 6.1 設計思想

- 日常入力時は**全額で入力**する（按分計算はしない）
- 設定画面で科目ごとの**事業使用割合**をスライダーで設定
- 損益計算書と所得税計算で**自動的に按分後の金額**を使用

### 6.2 対象科目

| 科目 | 按分の目安 |
|---|---|
| 地代家賃 (504) | 仕事部屋の面積 ÷ 家全体の面積 |
| 水道光熱費 (505) | 仕事時間 ÷ 1日の時間（例: 8h/24h ≒ 33%） |
| 通信費 (507) | 仕事での利用割合 |
| 車両費 (522) | 仕事での走行距離 ÷ 総走行距離 |

### 6.3 計算ロジック

```
按分後の経費 = floor(按分前の経費 × 事業使用割合%)

例: 家賃 80,000円/月 × 12ヶ月 = 960,000円
    事業使用割合: 50%
    按分後: floor(960,000 × 50%) = 480,000円
```

### 6.4 個別購入品の按分

PC・タブレット等の一点もので按分が必要な場合は、事業使用分の金額だけを入力し、メモに「○○円×事業○%」と記録する。

```
例: タブレット 50,000円、事業使用70%
→ 消耗品費 35,000円で入力
→ メモ: 「タブレット 50,000円×事業70%」
```

## 7. 損益計算書（PL）の仕様

### 7.1 計算ロジック

```
収入合計 = Σ（収入科目の貸方 − 借方）
経費合計 = Σ（経費科目の借方 − 貸方）× 家事按分率
事業所得 = 収入合計 − 経費合計（按分後）
```

### 7.2 表示

- 家事按分が設定されている場合、経費の「按分前」「按分後」の2列を表示
- 按分対象科目には按分率を表示（例: 地代家賃 (50%)）
- 事業所得の行に「家事按分適用後」のラベルを表示

## 8. 貸借対照表（BS）の仕様

### 8.1 構成

青色申告決算書の貸借対照表に準拠。

**資産の部:**
- 全資産科目（事業主貸を含む）
- 期首残高と期末残高を表示（期首残高がある場合）

**負債・資本の部:**
- 全負債科目（事業主借を含む）
- 元入金
- 青色申告特別控除前の所得金額

### 8.2 貸借等式

```
資産合計 = 負債合計 + 元入金 + 当期利益
```

差額が1円以上ある場合は警告を表示。

### 8.3 翌期繰越元入金（参考表示）

```
翌期繰越元入金 = 元入金 + 当期利益 − 事業主貸 + 事業主借
```

## 9. 所得税計算の仕様

### 9.1 計算フロー

```
事業収入 − 経費（按分後） = 事業所得
事業所得 − 青色申告特別控除 = 控除後所得
控除後所得 − 基礎控除 = 課税所得（所得税）
控除後所得 − 基礎控除（住民税） = 課税所得（住民税）
```

### 9.2 青色申告特別控除

| 種別 | 控除額 | 条件 |
|---|---|---|
| 65万円控除 | 最大650,000円 | 複式簿記 + e-Tax（または電子帳簿保存）+ 期限内申告 |
| 55万円控除 | 最大550,000円 | 複式簿記 + 紙提出 + 期限内申告 |
| 10万円控除 | 最大100,000円 | 簡易簿記 |

控除額は事業所得を上限とする（所得がマイナスの場合は0）。

### 9.6 復興特別所得税

```
復興特別所得税 = 所得税額 × 2.1%
```

### 9.7 住民税

```
所得割 = 課税所得（住民税ベース）× 10%
均等割 = 5,000円（道府県1,000 + 市町村3,000 + 森林環境税1,000）
住民税合計 = 所得割 + 均等割
```

## 10. 年度締め・繰越の仕様

### 10.1 処理内容

```
翌期の期首残高 = 当期の期末残高（以下のルールで調整）

資産科目（事業主貸以外）: そのまま繰越
負債科目（事業主借以外）: そのまま繰越
事業主貸 (104): 0にリセット → 元入金に振替
事業主借 (203): 0にリセット → 元入金に振替
元入金 (301): 期首元入金 + 当期利益 + 事業主借 − 事業主貸
収入・経費科目: 翌期は0からスタート
```

### 10.2 操作

設定画面の「年度締め」ボタンで実行。確認ダイアログ付き。再実行で上書き。

## 11. データ管理の仕様

### 11.1 保存先

- `localStorage` キー `aoiro-entries`: 全仕訳データ（JSON配列）
- `localStorage` キー `aoiro-settings`: 設定（JSON）

### 11.2 仕訳データの形式

```json
{
  "id": "一意のID",
  "date": "YYYY-MM-DD",
  "debitCode": "科目コード",
  "creditCode": "科目コード",
  "amount": 金額（整数）,
  "memo": "摘要（文字列）"
}
```

### 11.3 設定データの形式

```json
{
  "fy": 2026,
  "name": "事業者名",
  "deductionType": "65",
  "openingBalances": {
    "2026": { "101": 50000, "102": 800000, "301": 500000 }
  },
  "anbunRates": {
    "504": 50,
    "507": 40
  }
}
```

### 11.4 CSVエクスポート形式

```
日付,借方科目コード,借方科目,貸方科目コード,貸方科目,金額,摘要
2026-06-01,523,ソフトウェア利用料,205,クレジットカード,1000,Adobe
```

- BOM付きUTF-8（Excel対応）
- 摘要はダブルクォートで囲む
- CSVインジェクション対策: `=`,`+`,`-`,`@` 始まりのセルに `'` プレフィクス

### 11.5 CSVインポートのバリデーション

- 日付: `YYYY-MM-DD` 形式のみ
- 科目コード: 登録済みの科目のみ
- 金額: 正の整数のみ
- 借方 = 貸方の行はスキップ
- 同一内容（日付+科目+金額+メモ）の重複はスキップ

### 11.6 JSONバックアップ

設定 + 全仕訳データを1ファイルで保存・復元。バージョン番号付き。

## 14. 確定申告検証チェックリスト

本アプリを1年間運用し、確定申告に持っていけるか検証するためのチェックリスト。

### 14.1 日常運用

- [ ] 売上の入力（銀行振込 / 現金 / 請求書発行 / 売掛金回収）
- [ ] 経費の入力（各支払方法: 銀行振込 / 現金 / クレカ / 電子マネー / 個人立替）
- [ ] クレカ利用 → 引落日の処理が2回計上にならないか
- [ ] 電子マネーのチャージ → 利用の残高が合うか
- [ ] 事業主貸（生活費引出）の記録
- [ ] 個人立替（事業主借）の記録
- [ ] 固定資産の購入と減価償却の計上
- [ ] 月末に仕訳帳の件数・金額を確認

### 14.2 家事按分

- [ ] 按分率の設定（家賃・光熱費・通信費）
- [ ] 按分後のPLの数値が正しいか
- [ ] 按分後の税額が按分前より減っているか

### 14.3 年末の決算

- [ ] 未払いの経費がないか確認 → あれば未払計上
- [ ] 売掛金の残高が実際の未入金と一致するか
- [ ] クレジットカード残高が実際の未引落額と一致するか
- [ ] 減価償却費を計上したか
- [ ] 損益計算書の収入合計が実際の年間売上と一致するか
- [ ] 貸借対照表の現金・預金残高が実際と一致するか
- [ ] 貸借が一致しているか（警告が出ていないか）

### 14.4 確定申告書への転記

- [ ] 損益計算書 → 青色申告決算書の損益計算書（1ページ目）
- [ ] 貸借対照表 → 青色申告決算書の貸借対照表（4ページ目）
- [ ] 事業所得 → 確定申告書B 第一表「事業所得」欄
- [ ] 青色申告特別控除額 → 確定申告書B「青色申告特別控除額」欄
- [ ] 本アプリの所得税概算と、確定申告書の計算結果を比較

### 14.5 年度締め

- [ ] 年度締めボタンで翌年の期首残高が正しく作成されるか
- [ ] 翌年度に切り替えて、期首残高が反映されているか
- [ ] 事業主貸・事業主借が0にリセットされているか
- [ ] 翌期の元入金が正しいか（= 元入金 + 利益 + 事業主借 − 事業主貸）

## 15. 既知の制限事項

| 項目 | 状態 | 備考 |
|---|---|---|
| 消費税・インボイス | 未対応 | 免税事業者向け。課税事業者は別途対応が必要 |
| 不動産所得 | 未対応 | 事業所得のみ |
| 給与所得との損益通算 | 未対応 | 事業所得のみ |
| 固定資産台帳 | 未搭載 | 減価償却費の計算は手動。耐用年数表は未内蔵 |
| 減価償却の自動計算 | 未搭載 | 取得価額・耐用年数から償却額を算出する機能はない |
| 複数銀行口座 | 未対応 | 「普通預金」は1口座の前提 |
| 社会保険料控除等 | 未対応 | 所得税計算に含まれていない |
| 個人事業税 | 未対応 | 290万超の事業所得に課税される |
| データ同期 | なし | 端末間の同期はJSONバックアップで手動対応 |
| 税理士監修 | なし | 正式な申告前には税理士に確認してください |
